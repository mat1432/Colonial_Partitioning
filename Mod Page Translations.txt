### English
[b]1.34.5 ; This Mod Does Not Need to be Updated![/b]

This mod allows you to strip away all provinces any of your colonies has, that is in a colony charter outside their capital charter!
No more Brazil and Colombia completely colonizing Peru on an ironman save, you can just strip the lands to your possession and then unpause to create Colonial Peru!

This Mod is completely dynamic and should be compatible with just about every major map mod! Yes, it works on Random New Worlds, Anbennar, BT, Ante Bellum, etc.

This is a cheat mod, but I've included some settings so that it's less cheaty and more like gaming the system.


[h2]This Mod is:[/h2]
[b]NOT Achievement Compatible! (Modifies the checksum)[/b]
But will work on an Ironman save.


An event will pop at game start to specify if you want to enable cheaty options or not.

[h1]Features List:[/h1]
[h2]Diplomatic Actions:[/h2]
-> Can Instant Annex any of your colonies (requires enabling cheats at game start, or 'normal' on non-Ironman saves)
-> You can specifically set colonies to be protected from partitioning / instant annex
-> You can undo the partitioning of a colony (Only works on or after update 1.0.5)
- Will only undo lands that are your sovering provinces (Will NOT transfer lands owned by tributaries)

[h2]Partitioning:[/h2]
-> You can specify to skip any colonies that have not yet cored all provinces to at least a territory, (Uncored land does not form colonies)
-> You can specify to ignore the colony coring progress and just full core everything (not allowed on Ironman saves, and disabled by 'No Cheats' option at game start)

[h3]Undo Partitioning:[/h3]
-> Accessed by 'Undo Partitioning' Diplomatic Action under Influence Category
-> Will only transfer your soverign or colonial lands that were taken in a previous partitioning to the current colony.
-> Follows the same logic as regular partitioning, and all nations whose lands are taken from during this process will be flagged as partitioned.
-> Will only work on colonies partitioned after Update 1.0.5.
-> [b]The Undo process cannot affect active Players![/b]

[h2]Multiplayer Safety:[/h2]
-> By default, any colonies that were ever a player will be protected (Can be disabled)
-> Any colonies that were ever a player, if partitioned, will Always keep their cores (Including Territorial Cores) & gain Permanent claims on all lost land.

[h2]Install and Uninstall Compatibility:[/h2]
-> Forwards Compatability:
- The mod doesn't edit or replace any files, only adds files. It should be forwards compatible with every future update!
-> Backwards Compatibility:
- Same as above... The mod is compatible with prior versions, though don't go too far back because I'm not sure how far it is compatible with.
-> Save Compatibility:
- No need to worry about uninstalling, the mod only uses country, province and global flags which do not cause any save game damage upon uninstall.
You can safely use on any active save game and remove later if you wish.

[h2]AI[/h2]
[b]The AI will never use this mod or its features![/b]
If a game is saved with any menu open, the AI will close it without changes.
If a setup menu is given to the AI through save trickery, the AI will pick 'Normal' for the Cheats Menu and 'Singleplayer' for the MP menu.

Most of the code is for compatibility, fine-tuning, and ease of use.
At its core the code to partition colonies is like this:
[code]colonial_partition = { # decision
    potential = {
        ai = no
        any_subject_country = { is_colonial_nation = yes }
    }
    allow = { always = yes }
    effect = {
        hidden_effect = {
            every_subject_country = {
                limit = { is_colonial_nation = yes }
                every_owned_province = {
                    limit = {
                        NOT = { colonial_region = capital }
                    }
                    if = {
                        limit = { is_core = prev }
                        remove_core = prev
                    }
                    add_core = root
                    cede_province = root
                }
            }
        }
    }
}[/code]

### German
##Übersetzt von Thylon##
[b]1.34.5 ; Dieser Mod muss nicht aktualisiert werden![/b]

Diese Mod erlaubt es, alle Provinzen einer Kolonie zu entfernen, die sich in einer Kolonialregion außerhalb ihrer Hauptkolonialregion befinden!
Nie wieder werden Brasilien und Kolumbien Peru komplett kolonisieren, wenn du einen Spielstand hast. Du kannst das Land einfach in deinen Besitz bringen und dann die Pause des Spiels aufheben, um das Koloniale Peru zu erschaffen!

Dieser Mod ist komplett dynamisch und sollte mit so ziemlich jedem größeren Kartenmod kompatibel sein! Ja, es funktioniert mit Random New Worlds, Anbennar, BT, Ante Bellum, etc.

Dies ist ein Cheat-Mod, aber es gibt einige Einstellungen, so dass es weniger ein Cheat ist und mehr wie ein Teil des Spiels.


[h2]Dieser Mod ist:[/h2]
[b]NICHT Achievement-kompatibel! (Ändert die Prüfsumme)[/b]
Aber es wird bei einem Ironman-Spielstand funktionieren.


Beim Spielstart wird ein Ereignis angezeigt, in dem eingestellt werden kann, ob die Cheat-Optionen aktiviert werden sollen oder nicht.

[h1]Funktionen:[/h1]
[h2]Diplomatische Aktionen:[/h2]
-> Kann jede Kolonien sofort annektieren (erfordert die Aktivierung von Cheats beim Spielstart oder 'normal' bei Nicht-Ironman-Spielständen)
-> Sie können Kolonien speziell vor Partitionierung/Sofortannexierung schützen
-> Sie können die Partitionierung einer Kolonie rückgängig machen (funktioniert nur mit oder nach Update 1.0.5)
-> Macht nur Gebiete rückgängig, die zu den eigenen Provinzen gehören (überträgt NICHT Ländereien, die zu Tributstaaten gehören)

[h2]Partitioning:[/h2]
-> Sie können festlegen, dass alle Kolonien, die noch nicht alle Provinzen zu mindestens einem Territorium entkernt haben, übersprungen werden (Nicht Kernprovinzen können keine Kolonien bilden)
-> Sie können festlegen, dass der Kernprovinzen-Erstellung der Kolonie ignoriert wird und das alles zu Kernprovinzen gemacht werden (nicht erlaubt bei Ironman-Speicherständen und deaktiviert durch die Option "Keine Cheats" beim Spielstart)

[h3]Partitionierung rückgängig machen:[/h3]
-> Zugriff über die Diplomatische Aktion "Kolonieaufteilung rückgängig machen" in der Kategorie "Einflussnahmen".
-> Gibt nur Provinzen, die dem Spieler (der die Aktion ausführt) oder einer seiner Kolonien gehören und die bei einer früheren Partitionierung weggenommen wurden, an die aktuelle Kolonie zurück.
-> Folgt der gleichen Logik wie die normale Partitionierung, und alle Nationen, deren Land während dieses Prozesses genommen wird, werden als aufgeteilt gekennzeichnet.
-> Funktioniert nur bei Kolonien, die nach Update 1.0.5 partitioniert wurden.
-> [b]Der Rückgängig-Vorgang kann aktive Spieler-Nationen nicht beeinflussen![/b]

[h2]Mehrspieler Sicherheit:[/h2]
-> Standardmäßig werden alle Kolonien, die jemals von einem Spieler kontrolliert wurden, geschützt (kann deaktiviert werden)
-> Jede Kolonie, die jemals von einem Spieler kontrolliert wurden, wird, wenn sie geteilt wird, immer ihre Kernprovinzen behalten (einschließlich Territorialer Kernprovinzen) und permanente Ansprüche auf alles verlorene Land erlangen.

[h2]Kompatibilität bei Installation und Deinstallation:[/h2]
-> Vorwärtskompatibilität:
-> Der Mod bearbeitet oder ersetzt keine Dateien, er fügt nur Dateien hinzu. Sie sollte mit jedem zukünftigen Update vorwärtskompatibel sein!
-> Rückwärtskompatibilität:
-> Dasselbe wie oben... Der Mod ist kompatibel mit früheren Versionen, aber nicht zu weit zurück gehen, weil ich nicht sicher bin, wie weit es kompatibel ist
-> Spielstand-Kompatibilität:
-> Sie brauchen sich keine Sorgen um die Deinstallation zu machen, da der Mod nur Länder-, Provinz- und Globale Flaggen verwendet, die bei der Deinstallation keinen Schaden am Spielstand verursachen
Sie können diese Mod bei jedem aktiven Spielstand hinzufügen und später wieder entfernen, falls gewünscht.

[h2]KI[/h2]
[b]Die KI wird diesen Mod oder seine Funktionen niemals verwenden![/b]
Wenn ein Spiel mit geöffnetem Menü gespeichert wird, schließt die KI es ohne Änderungen.
Wenn der KI durch Speichertricks ein Setup-Menü gegeben wird, wählt die KI „Normal“ für das Cheats-Menü und „Einzelspieler“ für das Mehrspieler Menü aus.
